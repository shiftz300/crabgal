# Phase 1 桌面端验收提纲

> 验收对象：WebGAL 核心脚本语义、桌面 UI、场景流程与本地资源加载。
> 本轮不使用 hexz；资源来自 `projects/test-project` 的本地目录，并由 Bevy
> `AssetServer` 加载。验收前不需要打包或复制资源。

## 1. 启动前检查

在仓库根目录执行：

```bash
cargo fmt --all -- --check
cargo test --workspace
cargo clippy --workspace --all-targets -- -D warnings
cargo run -- dev projects/test-project
```

预期：workspace 测试全部通过，Clippy 无 warning；当前准确基线见
[`../webgal-compatibility/test-results.md`](../webgal-compatibility/test-results.md)。窗口打开后使用
`start.txt`，终端没有缺失场景、脚本解析或资源加载错误。Asset watcher 未启用的提示不影响脚本自身的热重载。

## 2. 首屏、变量和本地语音

1. 首屏应显示樱花背景、左侧立绘和文本框，姓名框宽度只包住“小夜”，不能铺满屏幕。
2. 正文应为“Phase 1 验收开始。数组第二项是 true。”，证明局部变量、全局变量、数组写入
   和正文插值走同一套表达式运行时。
3. 姓名应显示“小夜”而不是 `heroName`，证明 speaker 变量也被解析。
4. 应听到本地 `vocal/v16.wav`，音量为脚本指定的 65%；进入下一句后不应叠加旧语音。
5. 首次资源短暂加载时允许显示 Loading；完成后不应一直遮住画面。

## 3. 文本与公共参数

1. 推进到富文本句，显示内容应包含“样式文本”“蟹（かに）”和真实换行；`かに`
   必须使用合并字体正常显示，不能是方框、空白或缺字符号。
2. 当前 Phase 1 的约定是可读降级：style 标记不会泄漏到正文；真正的彩色 span 和 ruby
   排版留给 Phase 6，不在本轮冒充完成。
3. 下一段中立绘应向右偏移、轻微放大/旋转并出现局部 blur；背景切换并轻微放大/blur。
4. “这句只有条件成立时显示”必须出现，证明 `-when` 表达式生效。
5. “这句使用 -next”不应形成独立等待点，应直接落到下一句。
6. “这一句会在打字完成后自动前进”无需点击便进入选择，证明 `-notend` 生效。

## 4. Choice 外观与条件

选择界面应只出现四行，不应有标题或整块菜单底板：

- “隐藏选项”完全不可见；
- “禁用选项”可见但置灰，鼠标、Enter、数字键均不能进入；
- “本场景标签”“调用子场景”“切换场景”可选；
- 模糊只位于每个选项背后，选项间应保留清晰间隔；
- 上下方向键跳过禁用项，当前项有细边框/亮度反馈。

## 5. 三种分支目标

每种目标分别从标题页按 START 重开一次：

1. **本场景标签**：选择后显示“label 分支工作正常”，之后回到主场景收尾。
2. **调用子场景**：出现右侧访客和 `callScene` 说明；推进后自然返回主场景收尾，不能重新
   打开 Choice，也不能丢失原场景返回位置。
3. **切换场景**：切换到 `phase1_end.txt`，显示另一背景、右侧角色和 `changeScene` 说明；
   结束时直接去标题页，不能返回主场景。

## 6. end、标题页与重开

1. 主场景收尾应先淡出立绘和背景，再显示最后一句。
2. 最后一句推进后，舞台、对话、Choice、姓名框和控制栏全部消失。
3. 标题页只显示项目标题和 START；旧舞台不能透出。
4. 点击 START 或按 Enter/Space，应从 `start.txt` 第一段重新开始。
5. 局部变量必须重建；`-global` 变量允许跨 `end` 保留，但脚本重新赋值后结果仍应确定。

## 7. 回归检查

验收过程中顺带确认：

- 调整窗口大小后背景、立绘、Textbox 和 Choice 仍按 1920×1080 设计空间缩放，无拉伸；
- Q·SAVE / Q·LOAD 预览尺寸没有恢复成超大面板；
- Q·SAVE / Q·LOAD 浮窗区域应模糊其下方已经绘制的对白文字和控制栏，不只是背景/立绘；
  浮窗内的截图和文字本身必须保持清晰；
- Dialog 使用黑色遮罩，关闭后文字和局部 blur 正常恢复；
- Dialog 的背景模糊不得弱于 Textbox；当前参数分别为 36 和 30；
- 所有 UI 文字背后都有统一黑色背景阴影，浅色背景上仍清晰可读；
- 空 speaker 不显示姓名框；有 speaker 时姓名框不超过内容宽度；
- 连续快速推进不会卡死；脚本的 1024 action 上限已有自动测试保护；
- 所有图片、字体和语音均为本地路径，运行链路中没有网络请求和 hexz 依赖。

## 8. 本轮明确不验收

- 非 alpha 的 add/multiply/screen 合成渲染；当前只保留解析后的 blend 元数据，归 Phase 5；
- 真正的富文本样式 span、ruby 双层排版，归 Phase 6；
- BGM/SE 总线、Backlog、正式多槽存档、移动端 SafeArea 和输入适配；
- hexz 归档、加密、随机访问和发布打包。

发现问题时请记录“步骤编号 + 截图/录屏 + 预期/实际 + 是否稳定复现”。这样可以直接映射到
parser、core state 或 Bevy UI/render 三层，不需要从现象重新猜责任边界。
