# Phase 4 音频验收

## 启动

```bash
cargo run --release -- dev projects/test-project
```

点击 **START**，推进到选择界面，选择 **Phase 4 音频**。

## 按步骤验收

1. 进入子场景后，`v16.wav` 应在约 0.6 秒内从静音淡入并循环播放。
2. 等待 BGM 淡出提示句结束后再继续；下一句语音只应播放一次。点击控制栏上方第二个 Replay 图标，当前句语音应立即从头播放。
3. 打开 Backlog，点击任一带语音记录左侧的语音按钮，应只重播该条语音。
4. 推进到“一次性音效”一行：音效播放一次，结束后自动释放，不循环。
5. 推进到“循环音效”一行：带 `phase4-loop` id 的音效持续循环。
6. 在 CONFIG → AUDIO 拖动 MASTER VOLUME、VOICE VOLUME、BGM VOLUME 与 SOUND EFFECT VOLUME：对应声音应实时变化，滑杆松开后持久化。
7. 推进下一句后，`playEffect:none -id=phase4-loop` 应立即停止循环音效。
8. 最后一段开始时，BGM 应在约 0.6 秒内淡出；淡出期间动画不因显示器刷新率变化而改变时长。
9. 在 BGM 或循环音效存在时存档，再改变进度并读档：存档中的 BGM 与循环音效状态应恢复。
10. 返回标题后，剧情 BGM、语音与循环音效均不应残留。

## 通过标准

- 同一时间只有一个剧情 BGM；新 BGM 替换旧 BGM。
- 不带 id 的音效为一次性播放；带 id 的音效按 id 独立循环和停止。
- 所有脚本 volume 都先限制在 0–100%，再乘 MASTER 与对应分总线。
- 调整音量不会重启音频，淡入淡出不会固定占用高帧率生命周期。
- 缺少或损坏的音频资源只记录加载错误，不阻塞脚本状态机。
