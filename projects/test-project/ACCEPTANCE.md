# Feature Lab 分步验收

## 0. 启动基线

1. 运行 `cargo dev projects/test-project`。
2. 应先进入标题页；标题为 `crabgal Feature Lab`，背景是蓝天海边天台。
3. `CONTINUE` 在没有存档时禁用；点击 `START` 后进入 `start.txt`。
4. 首句出现时，背景、左侧立绘、textbox 和左侧姓名框应在同一入场阶段出现，不先闪 blur。
5. 姓名“绫”靠左排列，姓名框不会拉满屏幕；旁白时姓名框完全消失。

## 1. 对话与 textbox

1. 选择 `01`，按对白中的 `01-01` 至 `01-18` 顺序推进。
2. `01-01..04`：检查简写 say、官方 say、speaker 继承和 speaker 清空。
3. `01-05..10`：检查多行、转义、富文本、ruby、`<br>` 和内联等待。
4. `01-11..12`：mini avatar 只推动左边缘；隐藏后恢复；旁白无姓名框。
5. `01-13..14`：显式与单句自动 textbox 隐藏都必须同时处理姓名、正文和控制栏。
6. `01-15..16`：自动推进、`-next`、`-concat` 不产生重复 Backlog 行。
7. `01-17..18`：输入框有光标闪烁、统一按钮样式；确认后插值正确并恢复 textbox。

## 2. 流程、变量与选择

1. 选择 `02`，确认 `02-01..06` 的数字、浮点、布尔、字符串、数组、全局变量和条件结果。
2. 选择菜单中只应有三个可用分支、一个可见禁用项；隐藏项不可见。
3. 第一次选 `02-08`：进入右侧访客子场景，EOF 后回到 `02-10`。
4. 第二次重跑 `02` 并选 `02-09`：当前场景 label 后到 `02-10`。
5. 第三次重跑 `02` 并选 `02-12`：ChoiceTarget ChangeScene 进入新场景，再由 Action ChangeScene 返回主菜单。
6. 全程不得显示任何以“错误：”开头的句子，`comment` 也不得变成对白。

## 3. 舞台、变换与混合

1. `03-01..02`：背景必须保持 16:9，Hide/Show 间不残留旧图。
2. `03-03`：左中右三个锚点都位于舞台内，zIndex 次序正确。
3. `03-04..06`：中间立绘有清晰可见的约 9° 旋转；所有 transform 字段可见，四种 easing 平滑且与帧率无关。
4. `03-07`：四个互不重叠的对象从左到右为 Alpha/Add/Multiply/Screen，使用相同 authored alpha；Multiply 应随底色压暗，但轮廓透明边缘不得额外变黑。
5. `03-08`：四个测试对象全部消失；`none`、`-clear`、空内容三种退场语法都移除正确 id，日志不得请求 `placeholder.webp`。

## 4. 转场与遮幅

1. Intro 两页清晰显示；退出后不残影。
2. `04-01`：FilmMode 只覆盖 16:9 内容区；textbox、姓名框与 mini avatar 同步抬高，不被下方遮幅挡住。
3. `04-02..08`：依次检查 Instant、Fade、左右 Slide、Wipe、Dissolve、Crossfade。
4. 每次转场不得拉伸源图、闪白/闪黑或泄露视口外素材。
5. `04-09..10`：持久 enter/exit rule 只作用于指定 id，退出结束后实体被释放。

## 5. 动画与滤镜

1. `05-01..11`：逐个检查九个运动 preset 和两个命令别名，动画结束后回到稳定基态。
2. `05-12..17`：每个 film 都只作用于立绘并独立验收；Old Film 为原地闪动的随机噪点/细线，Dot Film 有粗网点、扫描线与中央凸起感，Glitch 快速随机，Godray 显著更慢，全部都不得卡住对话。
3. `05-18`：Remove Film 一次清空全部 bit。
4. `05-19..20`：四个 filter 字段可见且可完全还原。
5. `05-21`：Exit 最后一帧后移除立绘，重新生成时没有闪烁。

## 6. 粒子

1. `06-01..08`：依次检查轻/中/重雪、轻/中/重雨、萤火虫、落叶。三档雨线方向一致且互相平行；萤火虫明显大于雪粒；落叶使用叶片轮廓与暖色，不得看起来像雪。
2. 粒子始终裁切在 16:9 舞台内；调整窗口大小不改变设计空间中的尺寸和速度。
3. Heavy 预设仍应流畅；活动监视器中 CPU 不应因每粒子 ECS 实体显著上升。
4. `06-09` 后画面无粒子；`06-10` 再次验证全局清空无残留。

## 7. 音频与视频

1. `07-01`：WebGAL K 原 BGM《葉ノ舞》淡入并无缝循环；调整 BGM volume 跟手。
2. `07-02`：voice 完整播放；当前对白与 Backlog 的重播都从头开始。
3. `07-03..05`：双声道短提示完整播放，一次音效不循环，带 id 音效可调音量并准确停止。
4. `07-06`：视频第一次单击后才在右上角显示闪动小圆环，再次快速点击跳过；BGM duck 后恢复。
5. `07-07`：非阻塞视频约 1.1 秒后主动停止。
6. `07-08`：`skipOff` 禁止双击跳过。
7. `07-09..10`：`stopBgm` 与 `bgm:none` 都平滑淡出且不残留播放器实体。

## 8. UI、存档与工程能力

1. 按 `08-01..08` 的提示逐项打开 SAVE/LOAD/CONFIG/Backlog/Hide/Lock/EXTRA。
2. 所有全屏页共用同一 16:9 布局和 blur 强度；打开时 textbox 与姓名框同步淡出。
3. 直接打开 LOAD/SAVE 时就有与 CONFIG 一致的深色 blur 遮罩；SAVE 后留在当前页，槽位文字亮度一致，切页不闪黑白。
4. CONFIG 所有分页、滑动条、语言下拉、导入导出和返回按钮可用；关闭时内容与 blur 同步消失。
5. Backlog 退出无黑字、黑闪和残影；语音按钮图标与功能正确；触控板像素滚动平滑且不会被一个惯性像素误关。
6. EXTRA 左右 1:1，BGM 有可拖动进度条；CG/BGM 解锁内容可见。
7. 点击按钮或 Dialog 本身只执行 UI 动作，绝不同时推进对白；面板打开期间，推进、滚轮、Backlog 与快捷存档都被 input scope 阻止。
8. `08-02`：Q·SAVE 确认后仍停在 08-02；紧接着 Q·LOAD 也恢复到 08-02，确认按钮的同一帧点击不得泄漏到舞台。
9. 宽屏、窄屏、全屏和失焦下设计视口稳定；开发模式失焦不暂停。
10. 修改脚本保存后只热重载受影响场景，长期设置和存档仍在。

## 9. 结束流程

1. 主菜单选择 `09`；目标通过 ChoiceTarget ChangeScene 进入 `09_end`。
2. `09-01` 后推进，`End` 应停止音频、清空舞台和交互并平滑返回标题页。
3. 此时 `CONTINUE` 仅在存在有效存档时可用。

## 报错格式

请按“编号 + 现象 + 是否稳定复现”反馈，例如：

```text
06-04：雨线被横向拉伸；窗口 1280×720 和全屏都稳定复现。
```

这样可以直接定位到单一 Action/渲染路径并逐个修复。
